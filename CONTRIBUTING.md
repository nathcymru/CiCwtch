# Contributing to CiCwtch

## Contribution Model
CiCwtch is proprietary software. Contributions require prior approval.

Contributors may:
- Volunteer (approved contributors)
- Be contracted/procured for specific work

## Process
1. Open an Issue to discuss scope.
2. Await approval.
3. Fork and create a feature branch.
4. Submit a Pull Request.

## Standards
- Follow existing architecture patterns.
- Maintain readability and documentation.
- Ensure CI passes before submission.

By contributing, you agree that your work may be incorporated into commercial versions of CiCwtch.
